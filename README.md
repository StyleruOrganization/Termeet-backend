# Termeet-backend
## Разработка
- `docker-compose.yml` настроен именно для локальной разработки
- `docker-compose-dev.yml` - docker compose на тестовом сервере
- Миграции создавать внутри контейнера, тесты запускать также, потому что к БД можно достучаться только из контейнера
- Скоро будет настроено следующее, что через один Dockerfile собирается образ на сервер, а через другой для локальной разработки
- Тесты запускать внутри контейнера из корневой директории через команду python -m pytest -s <пусть к тесту>

## Запуск
### 1. Создать файл .env в корневой директории по аналогии с .env-example
### 2. Запустить docker compose
`docker compose build` - создать образ для контейнера backend  
`docker compose pull` - скачать образ для контейнера с БД  
`docker compose up --watch` - запустить в режиме разработки  
`docker compose up` - запустить
